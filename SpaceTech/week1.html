🗓️ Week 1 — Mission Timeline

Expand one day at a time. Focus on execution, not speed.

📘 Theory

Read Chapter 1 of Astrophysics for People in a Hurry. Focus on scale: cosmic time, billions of years, and why the universe is best understood through orders of magnitude.

💻 Python

Install Python. Open your editor. Write your first print() statement. Learn how code executes line by line.

🧭 Reference

Revisit the Big Bang video only if concepts feel abstract. No rewinding unless necessary.

📘 Theory

Read Chapter 2 of Astrophysics for People in a Hurry. Focus on the idea that light is a time machine — when you see stars, you are seeing the past. Pay attention to how distance in space is measured using light-years, not kilometers.

Try to internalize this idea: distance ≠ space alone, distance = time. This mental shift is foundational for all astronomy.

💻 Python

Learn variables, numbers, and strings. Understand how Python stores values in memory. Practice taking user input using input() and converting it into numbers.

Write small experiments: ask the user for a distance and print it back in a sentence. Focus on clarity, not cleverness.

🧭 Reference

If the idea of light-years feels abstract, revisit a short explainer on cosmic distance scales. Avoid watching multiple videos — one is enough.

📘 Theory

Read Chapter 3. Focus on energy, radiation, and motion. Notice how motion governs everything in the universe — from planets to galaxies to light itself.

Start thinking of the universe as a dynamic system, not a static picture. Nothing is still.

💻 Python

Learn loops (for and while). Understand repetition as a way to model motion and change over time.

Practice looping through numbers and printing values. Think of each loop iteration as a “moment in time.”

🧭 Reference

If loops feel confusing, slow down and rewrite examples by hand. Understanding loops now will save you months later.

📘 Theory

Read Chapter 4 of Astrophysics for People in a Hurry. Focus on how matter organizes itself into structures — stars, galaxies, clusters, and large-scale cosmic webs.

Think in terms of systems: individual objects matter less than how they interact. This systems-level thinking will later connect directly to programming and data science.

💻 Python

Learn functions. Understand the idea of inputs → processing → outputs. A function is a reusable unit of logic, just like physical laws are reusable across the universe.

Practice writing simple functions that take numbers as input and return calculated values. Keep names descriptive and simple.

🧭 Reference

If functions feel abstract, rewrite them in plain English first. Code clarity matters more than compactness at this stage.

📘 Theory

Read Chapter 5. This chapter introduces dark matter and the limits of our current understanding. Pay attention to what scientists admit they do not know.

This is an important mindset shift: not knowing is normal in science. Progress happens by asking better questions, not by having all answers.

💻 Python

Learn lists and basic indexing. Lists allow you to store multiple related values — just like astronomers store catalogs of stars or galaxies.

Practice creating lists, accessing elements, and looping through them. Focus on understanding how data is organized.

🧭 Reference

If lists feel confusing, visualize them as labeled boxes in order. Avoid jumping ahead to complex data structures.

📘 Theory

Do not read new chapters today. Instead, revisit your notes from Chapters 1–5. Focus on connecting ideas rather than memorizing facts.

Ask yourself: How do scale, light, motion, structure, and the unknown connect into a single picture of the universe?

💻 Python

Review all Python concepts learned this week: variables, loops, functions, and lists. Read your own code slowly and improve clarity.

Rename variables, simplify logic, and remove anything unnecessary. Clean code is a habit, not an afterthought.

🧭 Reference

If something still feels unclear, revisit notes — not tutorials. Your understanding should come from synthesis, not repetition.

📘 Theory

No new reading today. Instead, reflect on what surprised you most about the universe this week. Which ideas challenged your intuition?

Write a short paragraph (even outside the dashboard) summarizing how your view of space has changed.

💻 Python

Run your code one last time. Ensure everything works. Add comments explaining what each part does.

This prepares you for sharing and revisiting your work later.

🧭 Reference

Resist the urge to “learn more” today. Reflection is part of learning. Week 2 will build on this foundation.

🧠 Week 1 Intelligence Brief

Week 1 is designed to rewire scale perception. You are not learning facts — you are recalibrating how your brain handles magnitude, uncertainty, and abstraction.

Confusion here is structural, not personal. The universe does not map cleanly to human intuition, and neither does programming. That tension is the training.

Mastery is intentionally impossible this week. Orientation is the win.

📌 Important Notes

Read slowly, once. Re-reading too early creates illusion of understanding. Let concepts sit unfinished.

Code that runs imperfectly is superior to code that looks elegant but wasn’t written.

If something feels obvious on Day 5 that confused you on Day 2 — that delay is learning.
